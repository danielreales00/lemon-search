# Semantic recall — go/no-go bench (E5, ADR-0006)

Generated by `bench-runner -semantic` against the local Postgres, 24 hand-labeled NL/vibe queries from `bench/semantic-cluster.json`. Both arms run the **same** pipeline (intent ON, identical ranker); the only difference is `LEMON_FF_SEMANTIC` — the OFF arm is purely lexical, the ON arm embeds the query (all-MiniLM via Ollama) and adds the pgvector recall channel. So the delta isolates the **marginal lift of the vector channel** over the production lexical+intent baseline.

Ground truth is category/subcategory level (a vibe query should surface the right *type* of business): a query passes if any top-3 result's category/subcategory contains an expected token.

## pass@3

| arm | pass@3 |
|---|---|
| semantic OFF (lexical+intent) | 12/24 (50%) |
| semantic ON (+ vector recall) | 21/24 (88%) |

**+9 gained** (lexical miss → semantic hit), **0 regressed**.
