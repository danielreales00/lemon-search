# Ranking-formula comparison — 2026-05-28

Generated by `bench-runner -generate 300 -seed 42 -formula all` against the
local Postgres (22,568 rows). Each sampled business expands into exact/typo/
accent/partial query variants (plus 25 bare-category over_fire probes).
Every mode re-ranks the **same** retrieved candidate sets (Search +
ExactName are formula-independent), so the table isolates the ranking
formula. Modes differ only in `signal_formulas` (set in memory;
`config/ranking.yaml` is untouched):

- **literal** — spec default (`rating=lemon_score/10`, `distance=max(1-d/30mi,0)`)
- **bayesian** — `rating` smoothed over `google_rating` (0–5), prior C=20, m=4.3
- **decay** — `distance=exp(-d/decay_km[archetype])`

## Commentary

Best overall pass@3: **literal** (633/726 (87%)).
The findability set is dominated by exact/typo/accent/partial name recall, where the rating and distance formulas barely move the top-3 — the trigram/text score and the exact-name pin decide those, not rating smoothing or distance shape. The alternatives only re-order the *tail*.

**Default stays literal.** The spec contract (ADR-0004) fixes the 7-signal linear sum with literal formulas as the baseline; `bayesian`/`decay` are opt-in `signal_formulas` switches, not silent substitutions. Absent a decisive, intent-relevant win here, shipping literal honors the contract and keeps ranking explainable.
